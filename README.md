## ðŸ›  Development

### Prerequisites

- [WezTerm](https://wezterm.org/)
- [Bun](https://bun.sh/)

### Workflow

1. **Clone and Setup**:
   ```bash
   git clone https://github.com/MuXiu1997/dot-wezterm.git .
   bun install
   ```

2. **Build**:
   - `bun run build`: One-time compilation.
   - `bun run dev`: Watch mode for active development.

### Automated Deployment

The compiled Lua scripts are automatically deployed to the `dist` branch via GitHub Actions. This allows for a clean separation between the TypeScript source and the final Lua configuration used by WezTerm.

## ðŸ“‚ Project Structure

- `src/`: Source code (TypeScript)
  - `wezterm.ts`: Entry point.
  - `base-config.ts`: General terminal settings.
  - `command-palette.ts`: Command palette configuration.
  - `smart-right-click.ts`: Smart right-click behavior (Copy when text is selected, Paste otherwise).
  - `tmux-like/`: Keybindings and behavior mimicking tmux.
  - `tmux-nord-status/`: A Nord-themed status bar inspired by tmux.
  - `global.d.ts`: Type definitions for WezTerm and other globals.
- `dist/`: Compiled Lua files (generated by TSTL).
